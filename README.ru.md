## Основное назначение
Harvnyx — это не просто простой интерпретатор команд; это интегрированная среда выполнения команд. Она бесшовно сочетает внутренние команды, внешние исполняемые файлы Windows и команды, расширяемые через плагины, предоставляя пользователям единую точку взаимодействия.

## Техническая архитектура и основные модули
Программа состоит из нескольких основных классов, работающих вместе:
1. **Program.cs:** Логика взаимодействия и главный цикл программы.
   - Главный цикл: непрерывное чтение, разбор (поддерживается соединение нескольких команд операторами `&&`, `-and`, `-or`) и выполнение пользовательского ввода.
   - Расширенное взаимодействие:
      - Автодополнение по Tab: интеллектуальное и циклическое дополнение команд, подкоманд, опций и значений параметров.
      - Подсветка синтаксиса: цветовое различие разных элементов (основные команды, параметры, значения и т.д.) по мере ввода.
      - История и подсказки: запись истории команд и предоставление подсказок при вводе.
   - Динамическое приглашение: настраиваемое отображение текущей рабочей директории или сетевой информации (VPN/IP).
2. **CommandProcessor.cs:** Управляющий сегмент командной системы.
   - Маршрутизация команд: разбор пользовательского ввода и определение пути выполнения на основе имени команды: внутренняя команда, команда плагина или внешняя системная команда.
   - Разрешение конфликтов: четыре встроенных режима обработки (`Harvnyx`/`Windows`/`Mix`/`True`) для интеллектуального или интерактивного разрешения конфликтов имён между внутренними командами и внешними программами.
   - Разбор аргументов: поддержка сложных аргументов с кавычками и экранирующими символами.
3. **ConfigProcessor.cs:** Единый менеджер конфигурации и состояния.
   - Загружает и сохраняет пользовательские настройки из файла `config.ini`.
   - **Ключевые настройки:** режим обработки команд (`CommandInquiry`), содержимое приглашения (`additional`), очистка экрана при запуске (`ShellClear`).
   - **Системный мониторинг:** запускает фоновую задачу, которая периодически кэширует и быстро предоставляет информацию о загрузке/частоте ЦП и использовании памяти.
4. **CommandCompleter (`CommandProcessor.cs`):** Механизм дополнения и подсказок.
   - Поддерживает список всех доступных команд (встроенных, плагинов, внешних) вместе с метаданными их подкоманд и опций.
   - Динамически вычисляет текущий активный список команд на основе настроек в `ConfigProcessor`, предоставляя данные для дополнения и подсветки в `Program`.
   - ~~ИИ-копирайтинг — это круто~~
5. **CommandExecute (`CommandProcessor.cs`):** Исполнитель команд.
   - **`sudo`:** Имитирует повышение привилегий по образу Unix в Windows. Поддерживает запуск команд от имени другого пользователя, фоновое выполнение, кэширование временных меток и т.д.
   - **`shell`:** Управляет поведением самой оболочки, например, настройкой режима отображения приглашения и режима обработки команд.
   - **`tasks`:** Управляет фоновыми/передними задачами, запущенными Harvnyx – вывод списка, просмотр деталей или завершение задач.
6. **Система плагинов (интерфейс `ICommand` и `CommandManager`) (`CommandProcessor.cs`, `Program.cs`):** Ядро расширяемости.
   - Определяет интерфейс `ICommand`; любой класс, реализующий этот интерфейс, может быть загружен как команда-плагин.
   - `CommandManager` динамически сканирует и загружает `.dll`-файлы в текущей директории, автоматически регистрирует содержащиеся в них команды и обновляет систему дополнения.
7. **TaskManager (`Program.cs`):** Сервис хостинга задач. Отслеживает процессы и асинхронные задачи, запущенные Harvnyx, назначает им ID и позволяет пользователям управлять ими и контролировать их через команду `tasks`.

## Руководство по установке
### Системные требования
- **Операционная система:** Windows 7 SP1 / Windows 10 / Windows 11 (рекомендуется 64‑битная)
- **Среда выполнения:** [.NET 8.0 Runtime](https://dotnet.microsoft.com/en-us/download/dotnet/8.0) или выше
- **Зависимости:** Отсутствуют

### Способы установки

#### Способ 1: Установка через пакет
1. Перейдите на страницу [Releases](https://github.com/I-AM-SOLO-GetixPolaris/Harvnyx/blob/master/Program) и скачайте последнюю версию `Harvnyx.zip`
2. Распакуйте архив в любую директорию (например, `C:\Harvnyx`)
3. Зайдите в распакованную папку и дважды щёлкните `Harvnyx.exe` для запуска

#### Способ 2: Компиляция из исходников
1. Предварительные требования
   - [.NET 8.0 SDK](https://dotnet.microsoft.com/en-us/download/dotnet/8.0)
   - Git (опционально, для клонирования репозитория)
2. Шаги компиляции
```bash
# Клонируйте репозиторий
git clone https://github.com/NORTHTECH-Group/Harvnyx.git
cd Harvnyx

# Восстановите зависимости и соберите проект
dotnet restore
dotnet build -c Release

# Запустите
dotnet run --project Harvnyx.csproj
```
Скомпилированный исполняемый файл находится по пути `bin\Release\net8.0\Harvnyx.exe`

### Первоначальная настройка
- При первом запуске программа автоматически создаёт файл конфигурации `config.ini` в своей директории.
- Плагины команд (`.dll`) следует помещать в ту же директорию, что и программа; они будут загружены автоматически.
- Режим обработки команд по умолчанию — `Harvnyx` (использовать только внутренние команды). Его можно изменить с помощью `shell -i <режим>`. (Подробнее см. `Основное назначение`, раздел 2, подраздел 2.)
     - ~~**Защита от дурака**~~
        - **`Harvnyx`:** Использовать только внутренние команды
        - **`Windows`:** Использовать только команды Windows
        - **`Mix`:** Смешанное использование команд
        - **`True`:** Спрашивать пользователя перед выполнением команды

### Удаление
Просто удалите папку с программой. Записей в реестре не остаётся.

### Как внести вклад
#### 1. Сообщить об ошибке или предложить новую функцию

- Если нет, откройте новый Issue и выберите соответствующий шаблон:
  - **Отчёт об ошибке:** приложите информацию о системе, шаги воспроизведения, логи ошибок (используя вывод `print`).

#### 2. Улучшить документацию
- Документация находится в каталоге `docs/` (или в этом README).
- Исправление опечаток, улучшение форматирования, добавление примеров — всё это можно отправить через PR.
#### 3. Отправить код
##### Предварительные требования
- Знание C# и .NET 8.0
- Установленный локально [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
##### Рабочий процесс разработки
```bash
# Сделайте форк основного репозитория в свой аккаунт, затем клонируйте свой форк
git clone https://github.com/your-username/Harvnyx.git
cd Harvnyx

# Добавьте основной репозиторий как upstream (опционально, для удобной синхронизации)
git remote add upstream https://github.com/NORTHTECH-Group/Harvnyx.git

# Создайте ветку для функциональности
git checkout -b feature/название-вашей-функции

# Внесите изменения и убедитесь, что код компилируется
dotnet build

# Запустите (опционально)
dotnet run

# Зафиксируйте изменения (следуйте соглашению о коммитах)
git add .
git commit -m "тип: краткое описание"

# Отправьте в ваш форк
git push origin feature/название-вашей-функции
```
#### 4. Соглашения по коду
1. **Асинхронность:** Все потенциально блокирующие операции должны использовать `async/await` и поддерживать `CancellationToken`.
2. **Вывод сообщений:** Используйте класс `print` для вывода подтверждений/ошибок/предупреждений.
```C#
// Пример
print.error("команда/степень серьёзности", "описание проблемы", {код ошибки print.ErrorCodes});
// Необязательные степени серьёзности
public static class Severity
{
    public const string LOW = "Низкая";
    public const string MEDIUM = "Средняя";
    public const string HIGH = "Высокая";
    public const string CRITICAL = "Критическая";
    public const string FATAL = "Фатальная";
}
print.warning("команда/степень серьёзности", "описание проблемы");
print.success("команда/степень серьёзности(опционально)", "описание проблемы");
```
