# Harvnyx 0.1.80-Alpha.2 待修复问题清单（按缓急程度）

## 紧急优先级（P0 – 影响核心运行，可能导致崩溃或数据丢失）

### 1. 资源泄漏：`PerformanceCounter` 和 `ManagementObjectSearcher` 未释放
- **位置**：`ConfigProcessor.MainGetCPUAsync()`、`MainGetRAMAsync()`
- **问题**：未使用 `using` 语句，非托管句柄持续累积，长时间运行后可能导致系统资源枯竭。
- **修复**：为 `ManagementObjectSearcher`、`PerformanceCounter` 添加 `using`。

### 2. 后台进程输出重定向未读取，导致进程挂起
- **位置**：`CommandExecute.StartBackgroundProcess()`
- **问题**：设置了 `RedirectStandardOutput = true` 但从不读取，缓冲区满后进程阻塞。
- **修复**：设置 `RedirectStandardOutput = false`，或启动异步读取线程。

### 3. `TaskManager.KillTask()` 未检查 `HasExited` 即调用 `Kill()`
- **位置**：`TaskManager.KillTask()`
- **问题**：进程已退出时调用 `Kill()` 会抛出 `InvalidOperationException`。
- **修复**：先判断 `info.Process.HasExited`，若已退出则直接标记状态。

### 4. 热加载导致程序集锁定与内存泄漏
- **位置**：`CommandManager.ScanCommands()` 使用 `Assembly.LoadFrom()`
- **问题**：DLL 文件被锁定无法替换；多次加载后内存无法回收。
- **修复**：改用 `AssemblyLoadContext` 实现可卸载上下文，或禁用热加载，改为手动重启。

## 高优先级（P1 – 明显错误，影响用户体验或数据准确性）

### 5. 命令解析逻辑未处理引号内的操作符
- **位置**：`Program.ParseCommands()`
- **问题**：`echo "hello && world"` 中的 `&&` 被错误识别为命令分隔符。
- **修复**：在检测 `&&`、`-and`、`-or` 前，增加 `inQuotes` 状态判断。

### 6. `sudo` 密码缓存仅依赖时间戳，切换用户后仍有效
- **位置**：`CommandExecute._sudoTimestamps`
- **问题**：用户 `A` 验证密码后，用户 `B` 使用 `sudo -u B` 可能错误复用缓存。
- **修复**：缓存键应包含目标用户名（`当前用户+目标用户`）。

### 7. `FindExternalExecutable` 优先搜索当前目录，存在执行风险
- **位置**：`CommandProcessor.FindExternalExecutable()`
- **问题**：当前目录下恶意 `cmd.exe` 会优先于系统目录执行。
- **修复**：将 `Environment.CurrentDirectory` 移到最后搜索，或仅当命令包含路径分隔符时才搜索当前目录。

### 8. `ConfigProcessor` 中 WMI 查询失败时无降级方案
- **位置**：`MainGetCPUAsync()`、`MainGetRAMAsync()`
- **问题**：WMI 服务异常时返回错误字符串，导致抬头显示混乱。
- **修复**：增加备用方案（如 `PerformanceCounter` 获取频率、`GlobalMemoryStatusEx` 获取内存）。

### 9. 命令历史无限增长，内存占用无上限
- **位置**：`Program._commandHistory`
- **问题**：长期运行后历史条目过大。
- **修复**：限制最大条目数（如 1000），超出时移除最早记录。

---

## 中优先级（P2 – 性能下降或非致命缺陷）

### 10. 每30秒全量扫描 PATH 目录，性能开销大
- **位置**：`CommandCompleter.RefreshExternalCommands()`
- **问题**：网络驱动器或大量文件时 I/O 密集。
- **修复**：改用 `FileSystemWatcher` 监听系统 PATH 目录，或延长扫描间隔至 5 分钟。

### 11. CPU/内存采样过于频繁且内部延迟混乱
- **位置**：`ConfigProcessor.StartMonitoring()`
- **问题**：每 2 秒调用一次，内部又有 `Task.Delay(100)`，实际间隔波动。
- **修复**：统一采样间隔（如 3 秒），移除内部额外延迟。

### 12. ANSI 转义码在旧版 Windows 控制台显示乱码
- **位置**：全局 `COLOR_*` 常量
- **问题**：未启用 VT 模式时颜色失效。
- **修复**：启动时调用 `SetConsoleMode` 启用 `ENABLE_VIRTUAL_TERMINAL_PROCESSING`，或检测失败时禁用颜色。

### 13. 实时语法高亮每次按键都做正则解析
- **位置**：`Program.ParseInputForColoring()`
- **问题**：输入较长时卡顿。
- **修复**：仅当输入发生变化时重新解析，且使用 `Span` 或状态机替代正则。

### 14. 插件版本信息获取逻辑冗余且可能为空
- **位置**：`CommandManager.ScanCommands()`
- **问题**：同时使用反射和接口属性，仍可能返回 `null`，显示为 `"-"`。
- **修复**：统一从 `command.Version` 获取，若为空则尝试 `AssemblyInformationalVersionAttribute`。

## 低优先级（P3 – 代码质量与可维护性）

### 15. 静态类泛滥，无法进行单元测试
- **位置**：`CommandProcessor`、`CommandExecute`、`CommandCompleter` 等
- **问题**：全局状态强耦合。
- **修复**：重构为实例类，通过依赖注入管理（后续大版本）。

### 16. 超长方法 `HandleSudoCommand`（>200行）
- **位置**：`CommandExecute.HandleSudoCommand()`
- **问题**：难以理解和修改。
- **修复**：拆分为参数解析、密码验证、命令执行等私有方法。

### 17. 重复的密码读取和进程启动逻辑
- **位置**：`CommandExecute.ReadPassword()`、`ExecuteCommandAsUser()`、`StartBackgroundProcess()`
- **问题**：多处相似代码。
- **修复**：抽取公共方法 `PromptPassword()`、`CreateProcessWithUser()`。

### 18. 命名拼写错误：`COLOR_GEAY` → `COLOR_GRAY`
- **位置**：`Program.COLOR_GEAY`
- **修复**：更正为 `COLOR_GRAY`，并更新所有引用。

### 19. 配置文件解析忽略节标记 `[Settings]`
- **位置**：`ConfigProcessor.LoadConfig()`
- **问题**：用户手动添加其他节会导致后续键值被跳过。
- **修复**：实现标准 INI 解析（忽略节名，或仅读取指定节）。

### 20. 异常吞没无日志记录
- **位置**：多处 `catch { }`（如 `RefreshExternalCommands`、`StartBackgroundProcess`）
- **问题**：静默失败难以排查。
- **修复**：至少使用 `Debug.WriteLine` 或 `print.warning` 记录异常。
